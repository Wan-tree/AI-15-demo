# 方法与防止未来信息泄漏

## 1. 信息时间线

本项目默认的日频研究时间线是：

```text
t 日收盘/结算数据完成
    → 计算 t 日因子
    → 确定 t 日目标权重
    → 用执行日前已知的成交量和持仓量做准入检查
    → 公共执行层强制等待至少 1 个交易日
    → t+1 日用实际成交量和 OHLC 确认订单具备基本成交条件
    → t+1 日以结算价作为基础执行代理；不可成交则目标权重为 0
    → 获得 t+1 到 t+2 的具体合约收益
```

`configs/backtest.yaml` 的 `execution.signal_lag_days` 当前为 1，代码禁止设为 0。所有单因子和未来机器学习模型都共享该执行层。执行日成交量只用来判断预先提交的订单最后能否成交，不会回流到 t 日预测。结算价只是日频基础代理；如果以后改成次日开盘成交，就必须增加开盘价格、隔夜收益和日内收益拆分，不能只改一行 `shift`。

## 2. 主力选择

默认用 t-1 日持仓量选择 t 日主力合约。若用 t 日最终持仓量决定 t 日早晨交易哪个合约，会产生未来信息。

同时默认禁止交割月份回退，减少合约反复跳动。真实研究还应设置临近最后交易日的强制换月天数。

## 3. 连续价格

不同月份价格有天然价差。直接拼接结算价会把换月价差误认成单日收益。

本项目先计算“当前具体合约自己的日收益”，再把日收益连乘成指数化连续价格。连续价格用于因子，具体合约前瞻收益用于回测。

## 4. 特征和标签

- 特征只能使用当前及历史；
- 标签可以使用未来，但必须与特征列隔离，并从执行日而非信号日开始；
- 缺失标签不能填成零；
- 标准化、去极值和行业中性化参数只能在训练期估计；
- 不允许用全样本均值和标准差预处理测试期。

## 5. 数据划分

禁止随机拆分全历史。`configs/research.yaml` 已固定第一版边界：2020～2022 年训练、2023～2024 年验证、2025 年测试。所有模型必须使用同一边界：

```text
训练窗口 → 验证窗口 → 测试窗口 → 整体向前滚动
```

如果标签是未来 20 日收益且执行滞后 1 日，训练和验证边界还应预留至少 21 日隔离区，避免标签区间重叠。

由于团队已在框架调试中查看过 2025 年结果，它不能再被称为完全未观察的最终盲测。正式结论应在模型和策略冻结后，使用此前从未查看的新增数据只运行一次。

## 6. 成本

当前演示包括普通权重变化成本与主力换月成本，并固定输出 low、base、stressed 三档成本压力。压力测试解决“不能只展示一个有利成本假设”，但不能代替精确历史费率。真实研究仍必须确认：

- 手续费按手还是按成交额；
- 平今仓与平昨仓是否不同；
- 历史费率是否变化；
- 滑点以最小价位还是基点估计；
- 成交量是否足够容纳策略规模。

## 7. 结果解释

先比较毛收益与扣费后收益。如果毛收益良好但净收益消失，问题通常不是模型精度，而是换手过高或信号不可交易。
