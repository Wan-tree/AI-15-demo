# 架构与扩展指南

## 1. 分层原则

```text
外部文件/API
    ↓ connector
标准合约日行情
    ↓ validation
主力合约与连续序列
    ↓ features / labels
预测分数
    ↓ portfolio
目标权重
    ↓ backtest
每日收益、成本、持仓
    ↓ evaluation
图表和报告
```

每一层只依赖上一层的标准输出，不直接读取更上游的数据商格式。例如因子代码只认识 `settlement`，不应该知道某个供应商把结算价叫作什么。

## 2. 数据源扩展

新数据源应继承：

```python
from cn_futures_factor.data.connectors.base import DataConnector

class NewConnector(DataConnector):
    def load(self) -> pd.DataFrame:
        ...
```

`load()` 必须返回标准数据字典定义的字段。新增 API 连接器时，密钥从环境变量读取，不能写入源代码。

## 3. 因子扩展

1. 在 `features/` 新建函数；
2. 函数只使用当日及历史数据；
3. 在 `FACTOR_REGISTRY` 注册；
4. 在 `configs/factors.yaml` 启用；
5. 增加防未来信息测试。

因子函数不负责选主力、不负责下单、不负责保存文件。

## 4. 模型扩展

未来模型继承 `ResearchModel`：

```python
class LightGBMModel(ResearchModel):
    def fit(self, features, target):
        ...
        return self

    def predict(self, features):
        ...
```

模型层输出统一的 `prediction`。组合构建和回测无需知道模型类型。

机器学习扩展建议放在：

```text
src/cn_futures_factor/models/sklearn_model.py
src/cn_futures_factor/models/lightgbm_model.py
src/cn_futures_factor/research/walk_forward.py
```

需要机器学习依赖时运行：

```bash
pip install -e ".[ml]"
```

## 5. 回测扩展

当前回测是透明的日频向量化回测。未来可以增加：

- 品种级手续费表；
- 固定金额与成交额比例手续费；
- 合约乘数和整数手数；
- 保证金占用；
- 涨跌停不可成交；
- 夜盘交易日归属；
- 风险平价和板块中性；
- 事件驱动撮合。

不要在现有函数中不断增加大量 `if`。当执行逻辑发生根本变化时，新增一个实现并保持相同输出协议。

## 6. LLM 扩展边界

LLM 可以用于文献总结、研究日志、回测解释和非结构化信息提取，但不应直接负责基础 P&L、换月、风险约束或无校验下单。LLM 输出必须经过结构校验，并记录模型、提示词和时间。
