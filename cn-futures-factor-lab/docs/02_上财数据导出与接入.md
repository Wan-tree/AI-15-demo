# 上财数据导出与接入

## 1. 平台边界

访问入口：[上财实验数据资源平台](https://db.sufe.edu.cn/reservation/?tenant=shufelims)

平台提供虚拟应用、虚拟桌面、在线数据库、指定终端和部分落地数据库。不同资源具有不同许可与下载方式。本项目不自动登录、不保存 Cookie、不绕过访问控制，只处理你在授权环境中合法导出的本地文件。

## 2. 首次导出建议

不要第一次就下载全市场。先选择一个品种和一年日频数据，验证以下内容：

1. 是否包含所有具体月份合约，而不只是主力连续指数；
2. 合约代码是否带交易所后缀；
3. 成交量和持仓量单位是手、张还是千手；
4. 成交额单位是元、万元还是千元；
5. 结算价、前结算价是否同时提供；
6. 日期是否是交易日而非自然日；
7. Excel 是否有说明页、合并表头或单位行。

建议的必要字段：

```text
交易日期、交易所、品种、合约代码、开盘价、最高价、最低价、收盘价、
结算价、前结算价、成交量、持仓量、成交额
```

同时单独导出或整理：合约上市日、最后交易日、合约乘数、最小变动价位。

## 3. 保存原始文件

推荐命名：

```text
data/raw/sufe/<数据商>_<数据集>_<开始日期>_<结束日期>_<导出日期>.xlsx
```

例如：

```text
data/raw/sufe/vendor_contract_daily_20150101_20251231_20260721.xlsx
```

原始文件只读保存，不手动改列名、不删除说明行、不覆盖旧文件。若原始文件结构复杂，可以先保留原件，再另存一个清理版，并在研究日志中说明。

## 4. 配置字段映射

打开 `configs/field_mappings.yaml`。左侧是项目标准名，右侧改成导出文件中的列名。例如文件里写的是“交易日期”和“合约代码”：

```yaml
columns:
  trade_date: 交易日期
  contract: 合约代码
  settlement: 结算价
  volume: 成交量
  open_interest: 持仓量
```

如果成交量单位是千手：

```yaml
multipliers:
  volume: 1000.0
```

不要在 Python 源文件中硬编码中文列名。

## 5. 运行导入

```bash
cn-futures ingest \
  --input data/raw/sufe/你的文件.xlsx \
  --mapping configs/field_mappings.yaml \
  --output data/interim/sufe_contract_daily.parquet
```

指定 Excel 工作表：

```bash
cn-futures ingest --input 文件.xlsx --sheet 日行情
```

成功后会生成：

- 标准化 Parquet；
- 与输出文件同名的 `.manifest.json`；
- 数据检查摘要。

### 当前项目中的真实文件

2026-07-21 已接入：

```text
data/raw/sufe/sufe_all_contracts_daily_20200102_20251231_exported_20260721.csv
data/raw/sufe/sufe_vendor_main_contracts_daily_20200102_20251231_exported_20260721.csv
```

正式 Demo 使用第一份“全合约表”，配套配置为 `configs/field_mappings_sufe_all_contracts.yaml`：

```bash
cn-futures ingest \
  --input data/raw/sufe/sufe_all_contracts_daily_20200102_20251231_exported_20260721.csv \
  --mapping configs/field_mappings_sufe_all_contracts.yaml \
  --output data/interim/sufe_contract_daily.parquet \
  --allow-inactive-ohlc

cn-futures demo
```

必须注意：原始 `trade_date` 是 `YYYYMMDD` 整数，专用映射已设置 `date_format: "%Y%m%d"`。如果误用默认空格式，日期可能被解析为 1970 年。全合约表还包含部分“结算价有效、但未成交合约 OHLC 为 0”的记录；`--allow-inactive-ohlc` 只把它们降级为明确警告并保留结算价日期链，不代表这些记录可成交。

第二份供应商主力表只用于以后核对主力选择。它没有结算价，不能替代全合约表作为正式收益输入。

## 6. Manifest 的意义

Manifest 记录：

- 原始文件绝对路径；
- SHA-256 文件摘要；
- 导入时间；
- 行数和字段；
- 起止日期；
- 使用的映射配置。

如果同伴得到不同结果，可以先比较原始文件摘要，确认大家使用的是不是同一份数据。

## 7. 禁止事项

- 不把学校账号或密码写进代码；
- 不将原始授权数据提交 Git；
- 不把下载链接当作公开 API；
- 不使用脚本绕过预约、并发或下载限制；
- 不将授权数据发送给无权限成员。
