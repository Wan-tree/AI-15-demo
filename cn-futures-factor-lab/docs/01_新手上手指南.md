# 新手上手指南

## 1. 你正在搭建什么

这个项目把量化研究拆成七个可以单独检查的步骤：数据读取、质量检查、合约处理、因子、组合、回测和报告。模型只是其中一层，不是整个项目。

第一阶段先用合成数据确认工程能够运行，再接入上财平台导出的真实文件。这样可以把“代码问题”和“数据问题”分开处理。

## 2. 为什么使用 Conda

Python 项目经常因为包版本、系统 Python 或多人电脑环境不同而无法运行。Conda 环境相当于为本项目准备一个独立工具箱：

- 不影响电脑上的其他 Python；
- 明确使用 Python 3.11；
- macOS、Windows 和 Linux 可以使用同一份 `environment.yml`；
- 环境出错时可以删除后重建，不需要修复系统 Python。

## 3. 首次安装

打开终端，进入项目目录：

```bash
cd cn-futures-factor-lab
conda env create -f environment.yml
conda activate cn-futures-factor
```

`environment.yml` 已包含 editable 安装。为确保命令行入口是最新版本，也可以再执行：

```bash
python -m pip install -e ".[dev]"
```

运行检查：

```bash
cn-futures doctor
pytest
cn-futures demo
```

## 4. 每次重新打开终端

只需要：

```bash
cd cn-futures-factor-lab
conda activate cn-futures-factor
```

不需要每次创建环境或重新安装全部依赖。

## 5. 如何判断演示运行成功

终端应显示“演示运行成功”，同时 `artifacts/demo/` 中出现：

- `report.md`；
- `equity_curve.png`；
- `daily_returns.csv`；
- `positions.csv`。

报告必须明确说明结果来自合成数据。合成数据收益高低没有研究意义，只表示计算链路没有中断。

## 6. 常见问题

### `conda: command not found`

说明尚未安装 Miniforge/Anaconda，或终端没有初始化 Conda。完成官方安装后重新打开终端。

### 环境已经存在

使用：

```bash
conda env update -f environment.yml --prune
```

### 找不到 `cn-futures` 命令

先确认已经激活环境，再运行：

```bash
python -m pip install -e ".[dev]"
```

也可以直接使用：

```bash
python -m cn_futures_factor doctor
```

### 图表中文字显示异常

当前演示图使用英文标题，避免不同系统缺少中文字体。以后制作正式中文报告时，再统一配置项目字体。
