# 团队协作规范

## 1. Git 中可以提交什么

可以提交：

- 源代码；
- 配置模板；
- 文档；
- 测试；
- 不含授权数据的小型合成样本生成器。

不能提交：

- 上财数据库原始导出文件；
- 数据库账号、密码、Token、Cookie；
- 大型回测产物；
- Conda 环境目录；
- 个人绝对路径配置。

## 2. 分工建议

- 数据负责人：数据导出、字段字典、质量检查；
- 期货方法负责人：主力选择、换月、成本和交易规则；
- 因子负责人：因子定义、经济解释和防泄漏测试；
- 回测负责人：组合、绩效和敏感性分析；
- 文档负责人：研究日志、图表、汇报材料和复现实验。

同一个人可以承担多个角色，但每项输出应由另一位同伴复核。

## 3. 开发流程

1. 先写清楚要解决的问题和输入输出；
2. 修改代码；
3. 增加或更新测试；
4. 运行 `pytest`；
5. 运行 `ruff check .`；
6. 使用合成数据运行 `cn-futures demo --source synthetic`；
7. 在提交说明中记录方法变化和结果变化。

## 4. 实验记录

每个重要实验至少记录：

- 数据文件摘要或版本；
- 配置文件；
- Git 提交号；
- 训练、验证、测试区间；
- 使用的因子和模型；
- 成本假设；
- 主要指标；
- 已知限制。
