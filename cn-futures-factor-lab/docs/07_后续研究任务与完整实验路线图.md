## 0. 这份文档应该怎样使用

这不是一张“模型愿望清单”，而是一份从当前进度走到最终答辩的作业手册。团队应当按阶段执行，不要一开始就同时训练 LightGBM、LSTM 和 GNN。

每项任务都必须留下五类证据：

1. **代码**：放在 `src/`，不能只存在于 Notebook；
2. **配置**：记录数据、因子、模型、样本切分和成本参数；
3. **测试**：至少覆盖最容易出错的核心逻辑；
4. **产物**：预测值、持仓、指标、图表和日志；
5. **结论**：用三到十句话说明“做了什么、发现什么、有什么限制”。

一项任务只有同时满足以上五点，才算完成。程序能运行但没有测试，或有漂亮图片却说不清数据口径，都不算完成。

## 2. 先冻结研究问题

### 2.1 总研究问题

> 中国商品期货的历史价格、成交量、持仓量、期限结构和市场状态因子，能否预测未来可交易收益？LightGBM、XGBoost、神经网络或图神经网络，能否在严格样本外检验和交易成本后，稳定优于单因子、等权多因子和线性模型？

### 2.2 为什么不把“预测绝对价格”作为主任务

铜、螺纹钢和豆粕的价格尺度不同，价格水平通常也不是平稳序列。直接预测价格可能得到很低的误差，却不能说明模型知道涨跌，更不能说明策略能够盈利。

因此建议：

- **主任务**：预测未来收益 `future_return`；
- **主评价**：预测值与未来收益横截面排名是否一致；
- **主策略**：做多预测最高的一组、做空预测最低的一组；
- **辅助展示**：将预测收益转换为隐含价格：`预测价格 = 当前价格 × (1 + 预测收益)`；
- **辅助任务**：预测未来收益方向，但分类准确率不能替代组合回测。

### 2.3 建议的研究单元和标签

每一行样本表示：

```text
某交易日 t × 某商品品种 i
```

建议预先登记以下标签：

| 标签 | 定义 | 作用 |
|---|---|---|
| `target_return_1d` | t 至 t+1 可交易收益 | 最简单，噪声大，作为辅助 |
| `target_return_5d` | t 至 t+5 累计收益 | 推荐主标签 |
| `target_return_20d` | t 至 t+20 累计收益 | 中期稳健性检验 |
| `target_direction_5d` | 5 日收益是否大于 0 | 分类辅助任务 |
| `target_rank_5d` | 当日各品种未来 5 日收益百分位 | 排序/Ranker 拓展 |

主标签建议先定为 `target_return_5d`，原因是比 1 日收益更少受微小噪声影响，又比 20 日标签拥有更多独立时段。但最终必须结合真实数据长度确认。

推荐主实验每 5 个交易日产生一次信号并调仓，使标签期限和持有期容易对应。每天产生 5 日预测也可以，但相邻标签高度重叠；此时应采用 5 个错开的持仓子组合，并使用至少 5 日的隔离区和时间块统计方法。对零基础团队，第一版优先采用每 5 日调仓。

### 2.4 保守的信息与成交时间线

真实研究建议采用：

```text
t 日收盘/结算后获得完整行情
→ 计算 t 日因子和预测
→ t+1 日开盘价或结算价成交
→ 从 t+1 开始持有
→ 持有到预先定义的退出日
```

如果使用未来 5 日标签且以 t+1 日结算价进入，则应保证标签、持仓和退出价格对应同一条可交易合约链。项目还必须做“信号整体延迟 1 日”的敏感性检验。若延迟后效果完全消失，结论中必须明确说明信号对执行时点极度敏感。

### 2.5 预先登记假设

在第一次查看最终测试结果之前，写下：

- H1：至少一个基础因子的样本外 Rank IC 显著为正；
- H2：正则化线性模型优于简单等权多因子；
- H3：树模型能从非线性和因子交互中获得增量；
- H4：神经网络只有在样本量和正则化充分时才可能优于树模型；
- H5：GNN 只有在关系图有真实经济含义时才可能带来增量；
- H6：部分预测优势在交易成本后会明显衰减；
- H7：最复杂模型不一定是综合表现最好的模型。

如果结果不支持假设，也应如实报告。研究的目标不是证明机器学习一定胜出，而是判断在什么条件下它有没有增量。

## 3. 全团队必须遵守的研究规则

### 3.1 公平比较规则

所有模型必须共享：

- 同一数据版本；
- 同一品种池；
- 同一特征集合；
- 同一标签；
- 同一滚动切分日期；
- 同一缺失值规则；
- 同一预测到权重的转换；
- 同一交易成本；
- 同一评价指标。

只有这样，模型之间的差异才可能来自模型本身。

### 3.2 测试集隔离规则

- 最终测试区间在项目早期确定；
- 数据负责人可以检查测试期数据完整性，但模型负责人不能根据测试指标调参；
- 每个模型先在训练和验证折上完成选择；
- 模型配置冻结后才运行最终测试；
- 最终测试不允许反复试到满意为止；
- 若发现代码错误，修复后必须记录原因，并重新运行所有模型。

建议创建 `docs/test_access_log.md`，每次访问最终测试结果都记录：访问时间、人员、实验 ID、冻结配置、访问原因以及是否导致研究方案改变。正式研究按以下纪律执行：

1. 探索期只看训练折和验证折；
2. 候选模型、参数范围、策略规则和成本全部冻结；
3. 由两名成员共同执行一次最终测试；
4. 运行后无论好坏都不得再以测试表现为理由改模型；
5. 若因确定的程序错误必须重跑，旧结果标记为 `invalid`，修复说明和受影响实验必须保留，所有比较模型按同一修复重跑。

### 3.3 时间切分规则

不能随机打乱历史数据。建议实现滚动或扩展窗口：

```text
折 1：训练 ───── 验证 ─ 间隔 ─ 测试
折 2：    训练 ───── 验证 ─ 间隔 ─ 测试
折 3：         训练 ───── 验证 ─ 间隔 ─ 测试
```

如果标签使用未来 5 日收益并有 1 日执行滞后，训练与验证/测试之间建议至少留 6 个交易日的 `gap`；20 日标签加 1 日执行滞后建议至少 21 日。scikit-learn 的 `TimeSeriesSplit` 提供 `gap`、`test_size` 和 `max_train_size`，但项目还需按“交易日”而不是逐行切分，因为每天有多个品种。[官方说明](https://scikit-learn.org/stable/modules/generated/sklearn.model_selection.TimeSeriesSplit.html)

### 3.4 预处理规则

- 缺失率统计可看全数据结构，但填充值只能由训练期估计；
- 均值、标准差、分位点和 PCA 只能在训练集拟合；
- 验证集和测试集只能调用 `transform`；
- 树模型通常不强制标准化，但输入缺失规则必须统一；
- 线性模型和神经网络必须标准化；
- 横截面排名是当日可观察信息，可以按交易日计算；
- 不得使用未来日期的均值或波动率清洗历史数据。

### 3.5 随机性规则

- Python、NumPy、模型库和深度学习框架统一设置随机种子；
- 树模型至少记录一个固定种子；
- 神经网络至少运行 3 个种子，报告均值和标准差；
- 保存 Python、包版本、CPU/GPU 信息；
- 不应宣称不同平台或不同 PyTorch 版本可以做到完全逐位一致。PyTorch 官方也明确说明跨版本、平台和 CPU/GPU 的完全复现不保证，但可以通过固定各类随机源和确定性算法减少差异。[官方说明](https://docs.pytorch.org/docs/stable/notes/randomness)

### 5.1 Conda 环境和后续模型依赖怎样扩展

当前项目用 `environment.yml` 建立 Conda 环境，用 `pyproject.toml` 管理 Python 包。两者分工如下：

- `environment.yml`：锁定团队共同的 Python 大版本、基础科学计算库和开发工具；
- `pyproject.toml` 的核心依赖：所有流程都需要；
- `.[dev]`：测试和代码检查；
- 当前已有的 `.[ml]`：scikit-learn、LightGBM、XGBoost；
- 后续建议新增 `.[deep]`：PyTorch；
- 后续建议新增 `.[gnn]`：PyTorch Geometric 及其必要组件。

做到树模型阶段时，在现有 Conda 环境内执行：

```bash
conda activate cn-futures-factor
python -m pip install -e ".[ml]"
python -c "import sklearn, lightgbm, xgboost; print('ML dependencies OK')"
```

做到神经网络或 GNN 阶段时，再由环境负责人依据团队的操作系统和 CPU/GPU 情况确定 PyTorch 安装方式，并把确认过的版本写进可选依赖。不要让每位成员各自从网上选择不同的 CUDA 或 PyTorch 版本。建议流程：

1. 先在一台主实验机器上安装和验证；
2. 把版本、安装命令和硬件信息写入 `docs/environment_deep_learning.md`；
3. 在 `pyproject.toml` 新增 `deep` 和 `gnn` 依赖组；
4. 至少让另一台机器复现 CPU 小样本训练；
5. 再更新共享环境，执行 `python -m pip install -e ".[deep,gnn]"`；
6. 用小型张量、一次 MLP 前向传播和一次 GCN 前向传播作为环境验收。

基础依赖发生变化后，其他成员使用：

```bash
conda env update -f environment.yml --prune
conda activate cn-futures-factor
python -m pip install -e ".[dev]"
```

不要把多个不同项目都装进这个环境，也不要用系统 Python 安装模型库。GPU 不是完成项目的前提；第一版 MLP、LSTM 和小图 GCN 都必须支持 CPU 小样本运行。

### 5.2 阶段门：满足条件才能进入下一阶段

| 阶段门 | 通过条件 | 未通过时怎么办 |
|---|---|---|
| G0 工程可复现 | 全员 `doctor`、`pytest`、`demo` 通过 | 先修环境，不接真实数据 |
| G1 数据可相信 | 小样本字段、单位、20 行抽查、质量报告通过 | 回到映射或导出步骤 |
| G2 数据集无泄漏 | 标签、主力、品种池、时间切分测试通过 | 不允许训练模型 |
| G3 基准已冻结 | 单因子、等权、OLS/Ridge 结果可复现 | 不进入树模型 |
| G4 树模型已比较 | LightGBM/XGBoost 用公共协议完成 | 才能判断是否需要深度模型 |
| G5 深度模型有合格对照 | MLP 与树模型公平比较，至少 3 个种子 | 再决定是否做序列模型 |
| G6 图有意义 | 图统计、无未来边、随机图/无图对照准备好 | 不训练 GNN |
| G7 候选和策略冻结 | 候选模型、成本、仓位规则和阈值已登记 | 不打开最终测试集 |
| G8 最终证据完整 | 预测、回测、稳健性、失败记录、复现包齐全 | 不撰写强结论 |

阶段门的意义不是拖慢进度，而是防止把数据问题误认为模型能力。每次周会在任务板上明确当前所在阶段门。

# 第二部分：基准方法必须先于复杂模型

## 9. 阶段 3：建立完整基准线

基准不是“陪跑”。如果复杂模型不能稳定超过基准，正确结论就是复杂度没有产生价值。

### 任务 B000：无技能基准

实现：

- 预测收益恒为 0；
- 预测为训练期历史均值；
- 预测价格等于当前价格；
- 等权持有所有可用品种——只用于策略参照。

意义：证明模型至少优于“什么都不知道”。

### 任务 B001：单因子排序

对每个因子分别：

1. 每天做横截面排名；
2. 方向根据训练/验证期经济假设决定；
3. 多头最高组、空头最低组；
4. 多空各占 50% 毛敞口；
5. 使用公共成本和品种池；
6. 保存日预测、仓位和收益。

需要比较：

- 动量；
- 反转；
- 波动率；
- 流动性；
- 持仓变化；
- Carry/期限结构，数据允许后。

验收标准：每个单因子都有 IC、分组收益、净值、成本和分时期结果。

### 任务 B002：等权多因子

步骤：

1. 每个因子按日转成百分位排名；
2. 对方向相反的因子乘以 -1；
3. 对启用因子取等权平均；
4. 组合分数再做横截面排序；
5. 使用与单因子完全相同的组合规则。

至少做两版：

- `EW_ALL`：全部通过基础质量门槛的因子；
- `EW_CORE`：动量、Carry、波动率、流动性等预先定义核心因子。

不得根据最终测试期挑选因子。

### 任务 B003：OLS、Ridge，以及可选的 Lasso/ElasticNet

输入：同一日期×品种特征矩阵。  
目标：未来 5 日收益。

步骤：

1. 在每个滚动折内拟合预处理器；
2. 先跑普通线性回归；
3. 再跑 Ridge；
4. P1 扩展再跑 Lasso 和 ElasticNet；
5. 只用验证集选择 `alpha`，ElasticNet 还需选择 `l1_ratio`；
6. 冻结后预测测试折；
7. 将预测值作为统一组合分数。

Ridge 的 L2 正则有助于处理高度相关的因子。官方定义的目标函数是平方误差加 `alpha × L2` 惩罚。[scikit-learn Ridge](https://scikit-learn.org/stable/modules/generated/sklearn.linear_model.Ridge.html)

四种线性模型的意义不同：

| 模型 | 作用 | 本项目中的位置 |
|---|---|---|
| OLS | 不加正则的最基础线性关系 | 必做，但高相关因子下可能不稳定 |
| Ridge | L2 收缩，通常保留所有因子 | 必做的强线性基准 |
| Lasso | L1 收缩，可把部分系数压到 0 | 可选，用于稀疏选择对照 |
| ElasticNet | 混合 L1/L2 | 可选，兼顾高相关和稀疏性 |

起始候选：

```yaml
alpha: [0.01, 0.1, 1.0, 10.0, 100.0]
fit_intercept: true
```

Lasso/ElasticNet 可从以下小网格开始，若数据量或时间不足可以不做：

```yaml
alpha: [0.0001, 0.001, 0.01, 0.1]
l1_ratio_for_elastic_net: [0.1, 0.5, 0.9]
max_iter: 10000
```

验收标准：

- 所有线性模型使用完全相同数据；
- 验证集选参，不查看测试；
- 输出每折系数；
- 说明系数方向是否稳定；
- Lasso/ElasticNet 需报告非零系数数量；
- 预测和策略指标均保存。

### 任务 B004：基准冻结

在进入树模型前，形成基准表：

| 模型 | OOS Rank IC | OOS R² | 净 Sharpe | 最大回撤 | 换手 | 成本 | 参数量/复杂度 |
|---|---:|---:|---:|---:|---:|---:|---:|
| 最佳单因子 | | | | | | | |
| 等权多因子 | | | | | | | |
| OLS | | | | | | | |
| Ridge | | | | | | | |
| Lasso（若完成） | | | | | | | |
| ElasticNet（若完成） | | | | | | | |

这张表是所有复杂模型的共同参照，之后不能偷偷替换表现较差的基准。

# 第三部分：树模型

## 10. 阶段 4：统一模型实验器

### 任务 M000：模型协议和实验运行器

所有模型实现：

```python
model.fit(train_features, train_target, validation_data=...)
prediction = model.predict(test_features)
```

实验运行器负责：

1. 读取 `dataset_id`；
2. 读取固定切分；
3. 逐折拟合预处理器；
4. 拟合模型；
5. 产生验证和测试预测；
6. 运行同一组合构建；
7. 保存配置、环境、预测、指标和图表；
8. 对同一配置和种子产生可重复输出。

任何模型文件不得自己选择测试日期或交易成本。

### 任务 M001：LightGBM

为什么做：树模型能表达阈值效应和非线性交互，对表格因子常是强基准。

最小实现：

- `LGBMRegressor`；
- 回归未来 5 日收益；
- 验证集早停；
- 每折重新训练；
- 保存最佳迭代数；
- 保存 permutation importance 或 SHAP，解释只基于验证/测试预测后分析。

LightGBM 官方 Python API 提供 `LGBMRegressor` 和 `early_stopping` 回调，最佳迭代数会保存在模型属性中。[官方 API](https://lightgbm.readthedocs.io/en/stable/Python-API.html)

建议从小范围开始：

```yaml
objective: regression
n_estimators: 2000
learning_rate: [0.02, 0.05]
num_leaves: [7, 15, 31]
max_depth: [-1, 3, 5]
min_child_samples: [50, 100, 200]
subsample: [0.7, 1.0]
colsample_bytree: [0.7, 1.0]
reg_alpha: [0.0, 1.0]
reg_lambda: [0.0, 1.0, 10.0]
early_stopping_rounds: 100
```

不要做所有参数的笛卡尔积。第一轮固定多数参数，只探索树深、叶子样本和学习率；第二轮再精调。总试验数预先限制，例如每折 20～40 个候选。

验收：

- 每折有验证曲线和最佳迭代；
- 训练误差很好但验证恶化时能早停；
- 特征重要性在不同折中可比较；
- 与 Ridge 使用同一预测/策略评价。

### 任务 M002：XGBoost

最小实现与 LightGBM 相同，但使用 `XGBRegressor`。XGBoost 官方 sklearn 接口支持回归目标、评价指标和 early stopping。[官方 Python API](https://xgboost.readthedocs.io/en/stable/python/python_api.html)

建议起始范围：

```yaml
objective: reg:squarederror
n_estimators: 2000
learning_rate: [0.02, 0.05]
max_depth: [2, 3, 4]
min_child_weight: [10, 30, 60]
subsample: [0.7, 1.0]
colsample_bytree: [0.7, 1.0]
reg_alpha: [0.0, 1.0]
reg_lambda: [1.0, 10.0]
early_stopping_rounds: 100
```

验收：

- 固定 `random_state`；
- 验证集早停；
- 不用测试集决定迭代数；
- 保存训练时间和模型大小；
- 与 LightGBM 的差异只来自模型与其预先登记参数。

### 任务 M003：树模型消融

至少比较：

- 所有特征；
- 只有当前基础因子；
- 去掉期限结构；
- 去掉市场状态；
- 去掉横截面排名；
- 去掉成交量/持仓类；
- 打乱一个无关特征作为负对照。

目的不是把每次消融都当新模型调参，而是判断预测增量来自哪里。

### 任务 M004：树模型阶段结论

回答：

1. 树模型是否同时改善预测指标和净策略指标；
2. 增量来自哪些因子；
3. 是否只在某个年份或品种有效；
4. 是否因高换手抵消；
5. LightGBM 与 XGBoost 差异是否大于随机波动；
6. 若二者接近，应优先选择更稳定、运行更快或更容易解释者。

# 第四部分：神经网络

## 11. 阶段 5：从 MLP 到序列模型

只有在树模型和基准全部可复现后再安装深度学习依赖。建议以后在 `pyproject.toml` 增加 `deep` 可选依赖，不把大型依赖强塞给所有成员。

### 任务 N000：深度学习公共训练器

实现：

- CPU 和可选 GPU 自动选择；
- Python、NumPy、PyTorch 种子；
- 训练/验证 DataLoader；
- 训练损失和验证指标；
- early stopping；
- 保存最佳权重而不是最后权重；
- 梯度裁剪；
- 学习率调度可选；
- 3～5 个随机种子重复；
- 预测恢复到原始行索引。

验收：

- 同一平台、同一版本和种子下结果高度一致；
- 验证不改善时停止；
- 测试数据不进入训练循环；
- 训练中断后可从检查点恢复。

### 任务 N001：MLP 表格模型

MLP 与树模型使用同一行式特征，不引入序列。这是验证神经网络训练器是否正确的第一步。

建议起点：

```yaml
hidden_sizes: [[64, 32], [128, 64]]
activation: relu
dropout: [0.1, 0.3]
batch_size: [256, 512]
learning_rate: [0.0003, 0.001]
weight_decay: [0.0001, 0.001]
max_epochs: 200
patience: 20
loss: mse
```

作业：

1. 训练期拟合标准化；
2. 实现两层 MLP；
3. 添加 Dropout 和权重衰减；
4. 验证集早停；
5. 每折、每种子保存预测；
6. 报告均值和种子标准差；
7. 与 Ridge、LightGBM 使用同一策略。

如果 MLP 连 Ridge 都不能稳定超过，先排查训练和样本问题，不应马上增加网络层数。

### 任务 N002：序列数据集

LSTM 输入不再是一行，而是过去 L 日窗口：

```text
X.shape = [样本数, 序列长度, 特征数]
y.shape = [样本数]
```

实现注意：

- 每个序列只包含同一品种；
- 序列终点是 t；
- 标签从 t 之后开始；
- 不能跨品种拼接；
- 中间缺失日期需明确处理；
- 标准化器只在训练期拟合；
- 序列长度建议先比较 20 和 60；
- 样本索引必须保留 `trade_date + product`。

测试：在序列中人为放入特殊标记，确认没有读到 t+1。

### 任务 N003：LSTM/GRU

PyTorch 的 LSTM 在 `batch_first=True` 时输入可使用 `[batch, sequence, features]` 形式。[官方文档](https://docs.pytorch.org/docs/stable/generated/torch.nn.LSTM.html)

建议先做单向 LSTM，不做双向网络，因为双向结构容易在定义不严谨时混淆时间方向。

```yaml
sequence_length: [20, 60]
hidden_size: [32, 64]
num_layers: [1, 2]
dropout: [0.0, 0.2]
learning_rate: [0.0003, 0.001]
weight_decay: [0.0001, 0.001]
batch_size: [128, 256]
max_epochs: 200
patience: 20
```

模型：历史序列 → LSTM/GRU → 最后一个隐藏状态 → 线性输出未来收益。

验收：

- LSTM 和 GRU 使用同一序列数据；
- 3 个以上种子；
- 记录训练时间；
- 对比 MLP，判断“时间结构”是否有额外价值；
- 不因一次幸运种子宣称胜出。

### 任务 N004：Transformer 或 1D-CNN，可选

只有 LSTM 已正确完成且数据量足够时才做。

为了控制范围，只选择一个：

- 轻量 Temporal Transformer；或
- 1D-CNN/TCN。

必须限制参数量，并与 LSTM 使用相似参数规模。若样本量很小，跳过该任务比堆复杂模型更合理。

### 任务 N005：神经网络阶段结论

回答：

- MLP 是否超过树模型；
- LSTM 是否超过 MLP，说明序列信息是否有增量；
- 结果对随机种子是否敏感；
- 参数量增加是否带来稳定收益；
- 训练时间和解释成本是否值得；
- 神经模型在不同市场状态下是否更容易失效。

# 第五部分：图神经网络

## 12. 阶段 6：GNN 必须先证明“图”有意义

GNN 不是把普通表格换成另一个库。必须先回答：商品之间为什么连边？边在什么时候可观察？

### 12.1 GNN 启动门槛

只有同时满足以下条件才开始：

- 至少约 15～20 个长期可用品种；
- 有稳定的节点特征；
- 能构造不使用未来的关系图；
- MLP/LSTM 已有可复现基准；
- 团队能解释每种边的经济意义；
- 有足够时间完成无图和随机图消融。

不满足时，把 GNN 写入“未来工作”即可，不应做一个无法解释的装饰性模型。

### 任务 G001：定义节点

建议：每个商品品种是一个节点，例如 RB、CU、M。

节点 t 日特征：

- 当前因子；
- 最近一段收益序列的编码；
- 板块；
- 流动性；
- 市场状态。

节点标签：该品种未来 5 日收益。

### 任务 G002：构造三种图

#### 图 A：静态板块图

同属能源、黑色、有色、农产品、油脂、化工等板块的品种相连。

优点：可解释、不会使用未来数据。  
缺点：关系粗糙。

#### 图 B：滚动相关图

只使用 t 日以前固定窗口的收益相关系数：

1. 使用过去 60/120 日；
2. 对每个节点保留绝对相关最高的 k 个邻居；
3. 边权使用相关系数或绝对相关；
4. 每个交易日滚动更新；
5. 不允许使用全样本相关矩阵。

#### 图 C：经济关系图，可选

例如产业链上下游、替代品、共同原料。必须有可靠资料和固定规则，不能因为结果好再人工加边。

输出：

```text
trade_date | source_product | target_product | edge_type | edge_weight
```

测试：修改未来收益后，历史边必须完全不变。

### 任务 G003：图统计与合理性检查

每种图报告：

- 节点数；
- 边数；
- 平均度；
- 孤立节点；
- 边权分布；
- 网络随时间的变化；
- 代表性邻居是否符合常识。

如果一个品种几乎连接所有节点，或图每天剧烈变化，先修图而不是训练模型。

### 任务 G004：静态 GCN

PyTorch Geometric 的 `GCNConv` 接受节点特征、`edge_index` 和可选 `edge_weight`，输出新的节点表示。[官方文档](https://pytorch-geometric.readthedocs.io/en/latest/generated/torch_geometric.nn.conv.GCNConv.html)

最小结构：

```text
节点特征
→ GCNConv
→ ReLU + Dropout
→ GCNConv 或线性层
→ 每个节点的未来收益预测
```

建议：

```yaml
hidden_size: [16, 32, 64]
num_graph_layers: [1, 2]
dropout: [0.1, 0.3]
learning_rate: [0.0003, 0.001]
weight_decay: [0.0001, 0.001]
```

首版先使用当日节点特征和静态图，不要直接实现复杂时空 GNN。

### 任务 G005：GNN 必须做的消融

使用相同节点特征和训练器比较：

1. 无图 MLP；
2. 只有自环；
3. 随机图；
4. 板块图；
5. 滚动相关图；
6. 去掉边权；
7. 打乱节点对应关系。

若 GCN 只比无图 MLP 好，但与随机图无差异，不能说商品关系图有效。

### 任务 G006：时序 GNN，可选

在静态 GCN 已证明图有增量后，再考虑：

```text
每个节点的历史特征 → LSTM/GRU 编码
节点编码 + 当日图 → GCN/GAT
→ 未来收益预测
```

或者先图卷积、再时间编码。只能选择一个主结构，另一种作为拓展。

### 任务 G007：GNN 阶段结论

必须区分：

- 图结构有无增量；
- 时间编码有无增量；
- 参数量增加带来的增量；
- 结果是否只由少数枢纽品种贡献；
- 静态经济图和动态统计图哪个更稳定；
- 复杂度和计算成本是否值得。

GNN 未胜出也是有效结论：可能是品种数少、图不可靠、价格因子已经包含主要共振信息，或样本不足以估计复杂结构。

# 第六部分：预测如何变成策略

## 13. 阶段 7：统一预测到仓位的规则

### 13.1 主策略：横截面市场中性多空

主实验与 5 日标签对应，先采用“每 5 个交易日产生信号、次一交易日成交、持有 5 个交易日”的规则。对每个**预先定义的调仓日**：

1. 取得每个可用品种的预测收益；
2. 对预测值做横截面排名；
3. 做多最高 20%；
4. 做空最低 20%；
5. 多头权重和为 +0.5；
6. 空头权重和为 -0.5；
7. 总毛敞口为 1.0，净敞口接近 0；
8. 使用实际主力合约；
9. 按统一成本回测。

所有模型先使用这一个主策略。不能给表现差的模型低成本，给表现好的模型另选分位数。

如果团队之后改为“每天都产生未来 5 日预测”，必须把资金拆成 5 个错开的子组合：第 1 个子组合在第 1、6、11……个交易日调仓，第 2 个在第 2、7、12……个交易日调仓，依此类推，每个子组合占总资金的 1/5。这样能避免把 5 日预测错误地当作 1 日持仓信号。第一版不建议同时维护两种实现。

### 13.2 第二阶段策略改进

选定模型前，策略规则保持固定。模型比较结束后，才在验证集探索：

- 10%、20%、30% 多空分位；
- 1、5、10 个交易日调仓，且持有期和标签口径必须说明；
- 波动率倒数加权；
- 单品种最大权重；
- 板块中性；
- 目标组合波动率；
- 预测置信度阈值；
- 换手惩罚或缓冲区。

最终测试只能评估预先冻结的策略版本。

### 13.3 辅助策略：单品种时间序列信号

```text
预测收益 > 阈值：做多
预测收益 < -阈值：做空
其余：空仓
```

这是辅助实验，不能与横截面主策略混在一张表里直接比较，因为其风险暴露不同。

### 13.4 成本和可交易性

需要逐步完善：

- 品种级手续费；
- 按手和按成交额两种收费；
- 平今/平昨差异；
- 滑点，以最小跳动价位估计；
- 主力换月双边交易；
- 涨跌停不能成交；
- 流动性容量限制；
- 合约乘数和整数手数；
- 保证金和资金使用率。

至少报告三种成本情景：

```text
0.5 × 基准成本
1.0 × 基准成本
2.0 × 基准成本
```

# 第七部分：公平评价与稳健性

## 14. 阶段 8：预测指标

### 14.1 必须报告

| 指标 | 含义 | 注意 |
|---|---|---|
| MAE | 平均绝对预测误差 | 受收益尺度影响 |
| RMSE | 对大误差更敏感 | 不能单独决定策略好坏 |
| OOS R² | 相对零/历史均值基准 | 可能为负，不能隐瞒 |
| Pearson IC | 每日预测与未来收益线性相关 | 对极端值敏感 |
| Rank IC | 每日预测排名与收益排名相关 | 主指标 |
| ICIR | 平均 IC / IC 波动 | 需说明年化方式 |
| 方向准确率 | 预测涨跌是否一致 | 收益不对称时可能误导 |
| Top-Bottom spread | 最高组与最低组未来收益差 | 与策略最接近 |

不要只在全部行上算一个 pooled correlation。应先逐日计算 IC，再统计均值、标准差、正 IC 比例和各折结果。

### 14.2 按维度拆解

- 每个滚动折；
- 每年；
- 每个品种；
- 每个板块；
- 高/低波动状态；
- 趋势/震荡状态；
- 高/低流动性；
- 换月日/非换月日。

## 15. 阶段 8：策略指标

### 任务 V001：先统一绩效指标定义

当前 Demo 的 `src/cn_futures_factor/evaluation/metrics.py` 用“复合年化收益 ÷ 年化波动”计算名为 `sharpe` 的指标。这适合演示，但正式横向比较前必须拆开：

```text
算术 Sharpe = 日超额收益均值 ÷ 日收益标准差 × sqrt(252)
CAGR = 净值期末值 ^ (252 / 有效交易日数) - 1
年化波动 = 日收益标准差 × sqrt(252)
```

第一版可以假设无风险日收益为 0，但必须在报告中写明。正式结果同时报告 `sharpe_arithmetic`、`cagr` 和 `annual_volatility`，不要继续用 CAGR 除以波动替代标准 Sharpe。修改后需增加手算小样本测试，并重新生成所有基准结果；旧 Demo 数字不得和新口径直接比较。

必须报告：

- 毛收益和净收益；
- 年化收益；
- 年化波动；
- Sharpe；
- Sortino；
- 最大回撤；
- Calmar；
- 胜率；
- 换手；
- 交易成本；
- 多头和空头贡献；
- 品种和板块贡献；
- 最长回撤期；
- 不同成本情景；
- 模型训练和推理时间。

复杂模型必须同时报告计算成本和维护成本。只提高很少的 Sharpe、却显著增加依赖和不稳定性时，简单模型可能更值得选择。

## 16. 稳健性检验任务清单

### S001：时间稳健性

- 不同滚动折；
- 扩展窗口和固定窗口；
- 去掉表现最好年份；
- 危机/平稳时期；
- 最终封存测试期。

### S002：品种稳健性

- 逐个移除品种；
- 逐板块回测；
- 老品种与新上市品种；
- 高流动性品种子集。

### S003：参数稳健性

- 因子窗口前后变化；
- 多空分位变化；
- 交易频率变化；
- 模型关键参数邻域；
- 随机种子变化。

只有极窄参数点有效，通常是过拟合信号。

### S004：成本稳健性

- 0.5×、1×、2×成本；
- 加大滑点；
- 换月成本单独提高；
- 限制每日交易占成交量比例。

### S005：数据口径稳健性

- 收盘价与结算价；
- 持仓量与成交量主力；
- 不同换月缓冲；
- 不同品种池门槛；
- 真实合约收益与连续价格收益对照。

### S006：特征消融

- 一类因子整体移除；
- 只保留核心因子；
- 加入随机噪声；
- 打乱标签作为负对照。

打乱标签后模型若仍产生异常高表现，说明流水线可能泄漏。

### S007：统计不确定性

零基础阶段至少实现：

- 每折均值和标准差；
- 时间块 bootstrap 的 IC/收益置信区间；
- 神经网络多种子分布；
- 模型相对最佳基准的差值，而不只展示各自数值。

后续可增加 Newey-West/HAC、Reality Check 或多重检验控制，但必须先理解再使用。

# 第九部分：从探索到归纳再到最终结论

## 19. 探索阶段应该回答什么

探索顺序：

1. 数据是否可靠；
2. 哪些因子有基本预测关系；
3. 简单组合能否产生毛收益；
4. 成本后还剩多少；
5. 线性关系是否足够；
6. 树模型是否捕捉非线性；
7. 序列是否带来增量；
8. 图关系是否带来增量；
9. 增量是否跨时期、品种和成本情景稳定。

每完成一个阶段，写“阶段结论”，不要等最后一次性回忆。

## 21. 最终模型选择规则

在最终测试前，团队预先写下选择规则。可采用以下建议：

候选模型至少满足：

- 多数滚动折 Rank IC 为正；
- 相对最佳基准有稳定增量，而非只赢一个时期；
- 净收益不是完全由单一品种或年份贡献；
- 2×成本下不出现灾难性崩溃；
- 最大回撤和换手在可接受范围；
- 复杂模型的增量足以解释额外维护成本；
- 结果对相邻参数和随机种子不过分敏感。

可以在研究协议中预登记建议门槛，例如：

- Rank IC 相对最佳基准提高至少 10%；
- 净 Sharpe 提高至少 0.2；
- 最大回撤不超过最佳基准的 1.25 倍；
- 超过一半滚动测试折优于基准；
- 2×成本下累计净收益仍为正。

这些数字不是行业真理，应结合样本长度在测试前确定。关键是不能看完测试结果再修改门槛。

## 22. 不同结果对应的正确结论

### 情形 A：LightGBM/XGBoost 预测和策略都稳定胜出

可得结论：基础因子之间存在对未来商品期货收益有用的非线性与交互，树模型能提取这部分信息。

仍需限制：只在研究数据、时期、品种和成本假设内成立，不等于可以直接实盘。

### 情形 B：预测误差改善，但策略不改善

可得结论：统计预测增量没有转化为经济收益，可能因为排名改善不足、预测集中在不可交易小波动、换手过高或成本吞噬。

不能宣称“模型有效策略也有效”。

### 情形 C：树模型胜出，神经网络不胜出

可得结论：当前样本规模和表格因子更适合树模型，神经网络的额外参数没有得到充分支持。

### 情形 D：LSTM 胜过 MLP，但不一定胜过树模型

可得结论：序列结构存在增量，但非线性树模型仍可能具有更好的稳定性/成本比。

### 情形 E：GNN 胜过无图模型且通过随机图消融

可得结论：预先定义的商品关系图包含个体因子之外的信息。还需要说明是哪类边和哪些品种贡献。

### 情形 F：GNN 与随机图无差异

可得结论：性能可能来自参数量或正则化，而非经济关系图，不能把结果解释为“产业关系有效”。

### 情形 G：所有机器学习模型都不如等权因子

这是重要结果：在当前样本和成本下，简单透明的因子组合更稳健，复杂模型更容易过拟合。

### 情形 H：所有策略成本后失效

可得结论：因子可能存在统计预测性，但可交易性不足。应报告成本临界点，而不是隐藏成本。

### 情形 I：所有方法都没有可靠预测性

可得结论：当前数据、标签和特征不能支持稳定预测。继续堆模型不是合理下一步，应回到数据质量、因子经济逻辑和样本长度。
